## Requirement



1. Add a 'Reports' link to the Nav Bar for logged-in users. Clicking the name should display the 'Reports' page of the application.

2. Initially there will be only 1 selectable report. More will be added later.

3. The page will also show a list of previous reports that can be selected and viewed.

4. When the user selects the report a modal dialog box is to be shown showing date pickers for the start and end dates of the report. The start time is midnight on the start of the selected start day, and the end time is 11:59 at the end of the selected end day.

5. The dialog will also have 'Cancel' and 'Run Report' buttons.

6. The end date must be greater than or equal to the start day otherwise the 'Run Report' button should be disabled. If the dates are the same then this represents a 24-hour period from midnight to midnight on that date.

7. The end date can be the current date but no later.

8. The report period should be no more than 31 days, for example from the 1st of March 2026 to the 31st of March 2026. Disable the 'Run Report' button if the report period is too long.

9. When the user clicks the 'Cancel' button the dialog should be closed and no report is produced.

10. When the user clicks the 'Run Report' button, the dialog is closed and the report is produced in markdown format and stored in the database with it's creation date and time for later retrieval.

11. The new report is added to the list of reports shown in the UI, and also displayed in a new modal dialog and rendered as HTML. The report should be scrollable left/right and up/down so that the dialog does not go beyond the edges of the screen. The dialog will have a 'Dismiss' button to close the dialog. Selecting any report from the list of reports will retrieve the report from the database and render it in the dialog.

12. Follow the same style as the rest of the application.

13. This particular report should include the following sections:

	1. Project Tasks summary - a list of task summaries grouped per project, projects should be sorted alphabetically. Tasks within the project should be listed by completion date and should display the title and completion date only.

	2. Non-project Tasks summary - a list of tasks not associated with projects. Show the title and completion date and sort the list by completion dat.

	3. Cyclic Tasks Summary - a list of Cyclic Tasks with their name and completion date, sorted by completion date.

	4. Journal Summary - a list of Journal entries showing the title and the date of the entry, sorted by entry date.

	5. Trackers Summary - a list of completed trackers for the selected time period, showing the name of the tracker and a listed of dates that the tracker was completed, along with a summary per tracker showing 'N completed out of M days'.
